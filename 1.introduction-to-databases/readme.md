## Note for introduction about databases

### 1. អ្វីទៅជា មូលដ្ឋានទិន្នន័យ (Database)?
មូលដ្ឋានទិន្នន័យ គឺជាការប្រមូលទិន្នន័យដែលបានរៀបចំ ទុកដាក់ និងចូលប្រើតាមអេឡិចត្រូនិក។
មូលដ្ឋានទិន្នន័យត្រូវបានប្រើដើម្បីរក្សាទុក
និងគ្រប់គ្រងទិន្នន័យដែលមានរចនាសម្ព័ន្ធ និងគ្មានរចនាសម្ព័ន្ធ ក្នុងបរិមាណច្រើន ហើយទិន្នន័យនោះត្រូវបានប្រើដើម្បីគោលបំណងជាច្រើន រួមមាន៖
- ការរក្សាទុកទិន្នន័យ
- ការវិភាគទិន្នន័យ 
- និងការគ្រប់គ្រងទិន្នន័យ។

ឧទាហរណ៍៖ ស្រមៃថាអ្នកមានរូបថត ទំនាក់ទំនង និងកំណត់ចំណាំនៅលើ
ទូរស័ព្ទរបស់អ្នក (នោះគឺជាទិន្នន័យរបស់អ្នក)។ ជំនួសឱ្យការទុកវារាយប៉ាយគ្រប់ទីកន្លែង មូលដ្ឋានទិន្នន័យគឺដូចជាប្រអប់វេទមន្ត (ដូចបង្ហាញក្នុងរូបភាពប្រអប់) ដែលរៀបចំអ្វីៗគ្រប់យ៉ាងយ៉ាងស្អាត។ បន្ទាប់មក នៅពេលណាដែលអ្នកត្រូវការអ្វីមួយ អ្នកអាចរកវាបានយ៉ាងរហ័សដោយប្រើកុំព្យូទ័រ (ដូចបង្ហាញជាកុំព្យូទ័រនៅខាងស្តាំ)។

### 2. Database vs File Systems
ប្រព័ន្ធផ្ទុកទិន្នន័យឯកសារ និងប្រព័ន្ធទិន្នន័យ មានសារៈសំខាន់យ៉ាងខ្លាំងចំពោះការគ្រប់គ្រង និងរៀបចំទិន្នន័យប្រកបដោយប្រសិទ្ធភាពក្នុងការរចនាប្រព័ន្ធ។ ប្រព័ន្ធទាំងនេះផ្តល់នូវរចនាសម្ព័ន្ធសម្រាប់រៀបចំ ស្វែងរក និងរក្សាទុកទិន្នន័យក្នុងកម្មវិធី ខណៈពេលដែលធានានូវភាពអាចចូលប្រើបាន និងភាពត្រឹមត្រូវនៃទិន្នន័យ។ ប្រព័ន្ធទិន្នន័យ (Database) ផ្តល់នូវការគ្រប់គ្រងទិន្នន័យដែលមានរចនាសម្ព័ន្ធជាមួយនឹងសមត្ថភាពសួរចម្លើយកម្រិតខ្ពស់ ខណៈពេលដែលប្រព័ន្ធឯកសារ (File) គ្រប់គ្រងទាំងទិន្នន័យដែលមានរចនាសម្ព័ន្ធ និងទិន្នន័យដែលគ្មានរចនាសម្ព័ន្ធ។

ប្រព័ន្ធផ្ទុកទិន្នន័យដោយប្រើឯកសារ (File) សម្រាប់ការរចនាប្រព័ន្ធ
ប្រព័ន្ធផ្ទុកទិន្នន័យដោយប្រើឯកសារនៅលើកុំព្យូទ័រ ឬម៉ាស៊ីនមេ រក្សាទុកទិន្នន័យក្នុងឯកសារ ដាច់ដោយឡែកពីគ្នា។ ក្នុងប្រព័ន្ធថតឯកសារ និងថតរង ឯកសារនីមួយៗត្រូវបានរៀបចំទៅក្នុងថតមួយ និងត្រូវបានផ្តល់ឈ្មោះតែមួយគត់។

វិធីសាស្រ្តសាមញ្ញនេះដំណើរការបានល្អសម្រាប់ការរក្សាទុកទិន្នន័យទាំងដែលមានរចនាសម្ព័ន្ធ និងគ្មានរចនាសម្ព័ន្ធ ដូចជា ឯកសារកំណត់ហេតុ ឯកសារអត្ថបទ និងរូបថត។

- គុណសម្បត្តិនៃប្រព័ន្ធផ្ទុកទិន្នន័យដោយប្រើឯកសារ:
ភាពសាមញ្ញ៖ ងាយស្រួលក្នុងការអនុវត្ត និងគ្រប់គ្រង មិនតម្រូវឱ្យមានការកំណត់ដ៏ស្មុគស្មាញ។
តម្លៃសមរម្យ៖ សមស្របសម្រាប់តម្រូវការផ្ទុកទិន្នន័យតូចៗ ដោយគ្មានថ្លៃដើមខ្ពស់។

- គុណវិបត្តិនៃប្រព័ន្ធផ្ទុកទិន្នន័យដោយប្រើឯកសារ:
ភាពមានកំណត់ក្នុងការពង្រីក៖ មិនសមស្របសម្រាប់ប្រព័ន្ធធំៗ ឬតម្រូវការទិន្នន័យដែលកំពុងកើនឡើង។
គ្មានការគាំទ្រការទាញយកទិន្នន័យ(query)៖ អ្នកមិនអាចអនុវត្តការស្វែងរកកម្រិតខ្ពស់ដូចក្នុងប្រព័ន្ធទិន្នន័យបានទេ។
បញ្ហាភាពត្រឹមត្រូវនៃទិន្នន័យ៖ ការគ្រប់គ្រងភាពច្រើននៃទិន្នន័យ ឬទំនាក់ទំនងរវាងឯកសារអាចពិបាក។
